<h3>SSR with hidration strategy</h3>

<p>Initial HTML is generated on the server and sent to the client, Javascript bundle sent after to bring interactivity</p>
&nbsp;
<h4>Pros</h4>
<ul>
	<li class="small">Faster Initial Load</li>
	<li>SEO Friendly</li>
	<li>Better Performance for Low-end Devices</li>
</ul>
&nbsp;
&nbsp;
<h4>Cons</h4>
<ul>
	<li>Increased server costs</li>
	<li>Complexity in Development: 2 environments to care for</li>
	<li>Potentially frustrating UX due to TTI: the page may remain unresponsive until hydration finishes</li>
</ul>
