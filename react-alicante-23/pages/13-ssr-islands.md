<h3>SSR with islands strategy</h3>
<p>Involves rendering specific, interactive components (islands) on the client side while the rest of the page is rendered on the server.</p>
<h4>Pros</h4>
<ul>
	<li>Optimized Performance: Combined the benefits of SSR for initial load speed with CSR for island components</li>
	<li>SEO Friendly</li>
	<li>Improved User Experience: Users see the main content quickly while providing highly interactive islands</li>
	<li>Better Resource Management: focusing on what matters</li>
</ul>

<h4>Cons</h4>
<ul>
	<li>Development Complexity: Requires careful planning/architecting</li>
	<li>Fragmented Codebase: 4 dimensions: client/server envs & SSR/island code</li>
	<li>Initial Learning Curve</li>
</ul>
